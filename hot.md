- **`cold_start_base_multiplier = 0.65` chosen to raise cold-start ceiling to ≈ 90% (2026-05-15).** Base 65 + max positive modifiers 25 = 90. Old value 0.3 gave ceiling ≈ 55%. Requires `--historical --force` scorer re-run and web service restart to take effect on historical rows and API.
- **Cold-start tickers filtered from Telegram alerts at the notifier layer, not the DB query (2026-05-15).** Added Python filter in `get_qualifying_tickers()` after the SQL fetch, rather than adding `AND calibrated_score IS NOT NULL` to the SQL. Either approach is correct; the Python approach makes the filter visible and log-emitting.
- **`calibrated_cap=8.0` and `calibrated_scale=10.0` in `main.py:787` are still magic numbers (2026-05-15, follow-up observation).** The warm-start path (`min(abs(calibrated_score), 8.0) * 10.0`) uses hardcoded literals not read from config. A future change should migrate these into the same `confidence` config block (e.g. `confidence.calibrated_cap` and `confidence.calibrated_scale`). Deliberately out of scope for this change.
- **IQR p25/p75 thresholds for `signal_thresholds_raw` derived from full `scores_daily` dataset (n=30,974, 2026-05-15).** Per-regime sample sizes: ranging=9,326; trending=18,964; volatile=2,684. Values are hard-coded in `config/scorer.json` and dashboard-display only — no scorer re-run required. Refresh quarterly or after material distribution shift.
- **`DirectionBreakdown` exported for testability (2026-05-15).** Named export `// exported for tests` added in `TickerDetailPage.tsx` so F1/F2/F3 tests can render the component directly without full page setup.
- **`lightweight-charts` v5 over Recharts custom shapes for the price chart (2026-05-14).** Purpose-built candlestick library with canvas rendering; Recharts would require custom shape workarounds with no OHLCV model.
- **Separate `/api/price-chart` endpoint over embedding in `/api/snapshot` (2026-05-14).** Independent cache key orthogonality: snapshot is keyed by `(ticker, date)`, price-chart by `(ticker, range)`. Embedding bars in snapshot would force every snapshot refetch to re-fetch all OHLCV bars.
- **`DEFAULT_RANGE = "3M"` hardcoded as named constant, not read from config (2026-05-14).** UI interaction default, not a threshold. Changing it requires a frontend build and deploy — not a `config/web.json` edit.
- **`ALL` range cap = 5000 trading days in `config/web.json price_chart.range_days["ALL"]` (2026-05-14).** Single source of truth; exceeds any realistic backfill window so it effectively returns all rows on file.
- **Global header is zero-prop; page-scoped controls live in each page (2026-05-14).** `Header.tsx` renders only brand/nav/theme/signout and is mounted by every page. Anything page-specific (e.g. Ticker Detail's ticker picker + date picker + Load) renders in a sub-bar inside the page component below `<Header />`. Future pages must follow this split. See `DESIGN.md` §14.8.
- **Disabled nav placeholders use `<button disabled>`, not `<span>` (2026-05-14).** Required for screen-reader a11y — `aria-disabled` on a `<span>` is not announced. When a placeholder becomes a real page, swap the `<button disabled>` for a `<Link to="/...">` and the active-state logic already in `Header` works without changes.
- **Page-scoped controls sub-bar is not sticky (2026-05-14, deferred).** Only `<Header>` is `sticky top-0`. The Ticker Detail controls row scrolls away with the page. Two-layer sticky stacking would require either a single sticky wrapper or a hard-coded `top-[header-height]` value. Deferred — revisit if scrolling-to-switch-ticker proves annoying in daily use.

- **Per-timeframe data parity (commits 1-9)**: weekly + monthly now mirror daily's event tables (patterns, divergences, crossovers, swing_points, S/R, profiles) + score breakdown. Schema chose **Option A** (separate `scores_weekly` + `scores_monthly` tables keyed by week_start/month_start) over extending `scores_daily`. UI rendering chose **Option C** (per-timeframe score + categorical breakdown only — NO discrete BUY/SELL signal classification per timeframe; the merged daily signal remains the single decision the engine publishes).
- **`weekly_score_method` / `monthly_score_method` = `v2_8cat` in production (flipped 2026-04-29)**. v2_8cat loads weekly/monthly events from the mirror tables and routes crossovers→trend, divergences→momentum/volume, candlestick→cdl, structural→struct (cdl/struct weights currently 0.0, so the v2 scalar differs from v1 only via trend/momentum/volume getting weekly events). Flip was gated by a shadow acceptance gate run that PASSED with mean_delta=+0.004, std_delta=−0.006 — orders of magnitude inside thresholds. Reverting requires the same gate procedure in reverse (snapshot, flip back, re-historical, check).
- **scores_weekly/_monthly are denormalized snapshots** for query/UI — NOT in the scoring critical path. `merge_timeframes()` consumes the in-memory composite from `compute_*_score_breakdown()`.
- **Closed-period gate**: weekly closes when `scoring_date >= week_start + 7 days` (Sunday is in-progress; Monday closes the prior week). Monthly closes when `(scoring_date.year, .month) > (month_start.year, .month)`. Persistence skips in-progress periods; live merged signal uses partial bars.
- **Monthly candlestick scoring is permanently disabled** (decay-window mismatch — `_CANDLESTICK_WINDOW_DAYS=7` zeroes out monthly bars). `candlestick_score = None` in monthly breakdown regardless of v1/v2 mode.
- **`weekly_score` is in the LLM prompt context** (`ai_reasoner.py:803`), `monthly_score` is NOT. Flipping v2 will subtly shift Telegram blurb tone via the LLM input shift; mitigated by manual pre/post diff in the flip procedure.
- **Calibrator acceptance gate** uses distribution-level thresholds (max_mean_delta=5.0, max_std_delta=8.0 on calibrated_score across all tickers at a fixed scoring_date) rather than R²-based thresholds. Stores per-ticker values in baseline JSON to surface bipolar shifts. Three-tier output (PASS / PASS-with-WARNING ≥70% of threshold / FAIL).
- **Monthly timeframe added** (pre-parity): 3-way merge with regime-adaptive weights (trending 0.10d/0.50w/0.40m, ranging 0.60d/0.30w/0.10m, volatile 0.25d/0.45w/0.30m). When monthly is absent, remaining weights are renormalized to sum to 1.0.
- **Calibrator 17 features**: 6 category + 6 raw + 3 EMA spreads + weekly_score + monthly_score. Rolling ridge: window=365, lambda=0.1. Validated R²=0.1025.
- **Confidence base uses calibrated_score** (warm): `min(abs(cal), 8.0) * 10`. Cold start: `abs(final_score) * cold_start_base_multiplier` (config-driven via `confidence.cold_start_base_multiplier`, default 0.3, currently 0.65).
- **`final_score` column is always ±100** (merged composite). `calibrated_score` (≈ ±2–15%) is separate.
